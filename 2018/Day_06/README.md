# Day 6: Chronal Coordinates

## Part 1

This problem is just a series of [Breadth-First Searches](https://www.geeksforgeeks.org/dsa/breadth-first-search-or-bfs-for-a-graph/) (BFS) in order to find all coordinates which are part of each location's **dangerous area**. Starting at one of the given locations, we use a BFS to propagate the search outwards. A branch of the search terminates if it hits a coordinate which is **closer** to a different location, e.g.
```python
if any(manhattan_distance(coord, loc) <= distance for loc in locations if loc != start):
    continue
```

Using the example from the puzzle, the BFS starting at location `D` would propagate as follows. `d` represents coordinates within the `D` region and `*` represents coordinates where the BFS search stopped.

```

..........

.*ddd*..C.

```

The part 1 solution computes this BFS for all given locations, which returns the corresponding dangerous area. Then it returns the size of the largest area.

## Part 2 

We just do a standard BFS with the queue initialized with all locations in the input. We just add the following break condition
```python
if get_total_manhattan_distance(locations, coord) >= safe_distance:
    continue
```

So the algorithm starts at the locations in the puzzle input and continues to propagate until the total Manhattan distance exceeds the `safe_distance`. The algorithm returns the size of the computed area from the BFS.
